## 一、Tab 布局对比

### 官方结构
```
主面板：filepicker / load_raw / preview_crop / input_image / camera / simulation / exposure_control / enlarger / scanner / output
胶片：halation / couplers / grain / camera_diffusion
印放：glare / preflashing / diffusion
高级：spectral_upsampling / tune / special
设置：gui_config / display / 图层列表
```

### 我们的结构（当前，4 个 Tab）
```
主面板：预设 / 导入图像 / 输入图像 / 胶片与相纸 / 印刷色调 /
        微调 / 曝光控制 / 裁剪与放大 / 显示 / 图层列表
效果：halation / grain / 颗粒裁剪预览 / glare / camera_diffusion /
      diffusion / preflashing / couplers
高级：load_raw / camera / scanner / output / spectral_upsampling / special
设置：gui_config
```

底部操作栏（所有 Tab 可见）：自动预览 / 负片模式 / 扫描相纸模式 / 预览 / 扫描 / 保存
Tab 栏右上角：⇆（对比模式）/ ↺（恢复默认）/ ？（帮助）

### 差异与风险评估

| 差异 | 官方位置 | 我们的位置 | 风险 |

| `load_raw`（RAW 参数） | 主面板 | 高级 Tab | ✅ 有意移动，已讨论确认 |

## 三、widget_specs.py 参数范围对比

以下列出我们修改了 min/max/step 的参数，以及修改是否合理。

### 有意收窄范围（基于物理依据）

| 参数 | 官方范围 | 我们的范围 | 依据 |
|---|---|---|---|
| `exposure_compensation_ev` | min=-100, max=100 | min=-5, max=5 | 相机实际 EV 范围，±100 无意义 |
| `print_y_filter_shift` | 无 max | min=-50, max=50 | Kodak CC 实际范围 |
| `print_m_filter_shift` | 无 max | min=-50, max=50 | Kodak CC 实际范围 |
| `camera_lens_blur_um` | 无 max | max=15 | 镜头模糊物理上限 |
| `camera_diffusion_filter_spatial_scale` | 无 max | max=4 | 合理倍率上限 |
| `diffusion_filter_spatial_scale` | 无 max | max=4 | 合理倍率上限 |
| `scan_lens_blur` | 无 max | max=3 | 扫描仪模糊物理上限 |
| `scan_unsharp_mask` | 无 max | max=3 | 锐化合理上限 |
| `grain.particle_area_um2` | step=0.2 | step=0.05, max=2 | ISO 3200 约 1.5，加了上限 |
| `grain.blur` | 无 max | max=2 | 颗粒模糊合理上限 |
| `grain.blur_dye_clouds_um` | 无 max | max=3 | 染料云模糊合理上限 |
| `preflashing.exposure` | 无 max | max=0.5 | 预闪超过 0.3 已明显降低对比度 |
| `display.preview_max_size` | min=128, max=1024 | min=256, max=2048 | 扩大了上限，支持更高清预览 |
| `upscale_factor` | min=0.0 | min=0.5, max=3 | 加了合理上限 |

**风险评估：** 所有收窄都是 GUI 层的显示限制，不影响后端计算。用户仍可通过直接输入数值突破限制（FloatEditor 不强制 clamp）。但 SliderFloatEditor 会被限制在范围内。

### 新增了 label（官方只有 tooltip 无 label）

官方 `glare`、`halation`、`couplers`、`grain` 的大多数字段只有 tooltip，没有 label（显示字段名）。我们给所有字段加了中文 label。这是纯改进，无风险。

### 标签改名（可能引起混淆）

| 官方标签/字段名 | 我们的标签 | 风险 |
|---|---|---|
| `camera_lens_blur_um`（镜头模糊 um） | 镜头柔化 | ⚠️ 去掉了单位 um，用户不知道这是微米值 |
| `camera_diffusion_filter_spatial_scale`（空间缩放） | 扩散范围 | ⚠️ 含义有偏差，"空间缩放"是倍率，"扩散范围"像是绝对值 |
| `diffusion_filter_spatial_scale`（空间缩放） | 扩散范围 | 同上 |
| `scan_lens_blur`（扫描镜头模糊） | 输出柔化 | ⚠️ 去掉了"镜头"和单位，但 tooltip 有说明 |
| `scan_white_correction` | 输出白点校正 | ✅ 合理，"扫描"改"输出"更通用 |
| `scan_black_correction` | 输出黑点校正 | ✅ 合理 |
| `scan_unsharp_mask` | 输出锐化 | ✅ 合理 |
| `grain.particle_area_um2`（颗粒面积 um²） | 颗粒大小 | ⚠️ 去掉了物理单位，用户不知道这是 um² |
| `grain.blur_dye_clouds_um`（染料云模糊 um） | 颗粒扩散 | ⚠️ 去掉了单位，且"颗粒扩散"与 `blur`（模糊 sigma）容易混淆 |
| `couplers`（染料耦合剂） | 色彩交叉（高级） | ⚠️ 完全改变了作者术语，"染料耦合剂"是化学术语，"色彩交叉"是泛化描述 |
| `enlarger`（放大机） | 印刷色调 | ⚠️ 官方 EnlargerSection 包含 Y/M 滤镜，我们的"印刷色调"只是其中一部分 |
| `print_y_filter_shift` | 暖 ←→ 冷 | ✅ 方向已实测修正，直觉友好 |
| `print_m_filter_shift` | 品红 ←→ 绿 | ✅ 方向已实测修正，直觉友好 |

## 四、params_mapper.py 映射完整性

两版 params_mapper 逐行比对，**映射完全一致**，无遗漏、无错误。

唯一差异：我们的版本硬编码了以下性能参数：
```python
params.settings.use_enlarger_lut = True   # 官方也是 True（params_mapper 里）
params.settings.use_scanner_lut = True    # 官方也是 True（params_mapper 里）
params.settings.lut_resolution = 17       # 官方也是 17
params.settings.use_fast_stats = True     # 官方 params_mapper 里也是 True
```
这四项官方 params_mapper 里也是硬编码的，我们只是保持一致。

## 五、官方 preview 按钮 tooltip 的重要信息

官方 `widget_specs.py` 里 preview 按钮的 tooltip：

> "在小尺寸预览上运行模拟，**颗粒、halation、模糊、锐化蒙版均关闭**（柔光滤镜保持开启）"

我们的 tooltip：

> "快速预览效果（约 2-4 秒）。日常调参用这个，不需要每次都扫描。"

**⚠️ 我们的 tooltip 丢失了关键信息：预览模式下哪些效果被关闭。** 用户不知道预览里看不到颗粒和 halation，可能误以为这些效果没有生效。

## 六、官方 scan_for_print 功能

官方 `SimulationSection._apply_scan_for_print_mode()` 实现了一个重要的联动逻辑：

- 开启"为印放扫描"时：自动开启白点校正 + 黑点校正，自动关闭眩光
- 关闭时：恢复之前的状态

我们的版本完整保留了这个逻辑（`widget_sections.py` 里的 `_apply_scan_for_print_mode`）。✅
